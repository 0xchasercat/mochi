## Goal

Set up [Changesets](https://github.com/changesets/changesets) for independent per-package semver bumping per PLAN.md §15.8. After this lands, every PR that ships a user-visible change requires a changeset (CI enforces), and merging an accumulated batch triggers a release PR that publishes to npm with provenance signing.

## Out of scope

- Actual release execution. This task ships the harness, not a release. The first real release happens when phase 0.1 lands.
- npm token / OIDC repository configuration in GH Actions secrets — the user does this in the repo's Settings → Secrets and variables → Actions. Document the required secret name (`NPM_TOKEN`) in a comment at the top of `release.yml`.
- Custom release notes formatting beyond changeset defaults.
- Automated git-tag creation (changesets handles this).
- Linking more than the `net` ↔ `net-rs` pair. Explicit user-visible packages stay independent.

## Implementation notes

- `@changesets/cli` v2.27+ supports modern monorepos. For `bun changeset init`, run it once to scaffold `.changeset/`. The generated `.changeset/config.json` will be edited to add the linked packages.
- Linked packages format: `"linked": [["@mochi.js/net", "@mochi.js/net-rs"]]`.
- The `release.yml` workflow uses `changesets/action@v1` (the canonical action). Setup-bun runs first; the action handles PR opening/updating and publish.
- The provenance flag is set at the npm level: in `release.yml`, set `NPM_CONFIG_PROVENANCE: true` as an env var on the publish step. Combined with `id-token: write` permission, OIDC works automatically.
- `bunx --bun` ensures Bun runs the changeset binary natively, not via Node shim — keeps the runtime invariant.
- The initial-foundation changeset goes in `.changeset/initial-foundation.md` with frontmatter:
  ```
  ---
  "@mochi.js/core": patch
  "@mochi.js/consistency": patch
  "@mochi.js/inject": patch
  "@mochi.js/net": patch
  "@mochi.js/net-rs": patch
  "@mochi.js/behavioral": patch
  "@mochi.js/profiles": patch
  "@mochi.js/harness": patch
  "@mochi.js/cli": patch
  ---

  Initial v0.0.1 claim release with placeholder exports. Surface lands incrementally per PLAN.md §14.
  ```

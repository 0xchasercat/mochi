## v0.9.0 — 2026-05-10

`@mochi.js/core` 0.9.0 + `@mochi.js/cli` 0.2.7 + `@mochi.js/harness` 0.1.12 + `@mochi.js/behavioral` 0.1.6.

- **`mochi.connect(opts)` — attach to a CDP browser mochi did NOT spawn.** Public API for BrowserBase, Browserless, dockerised Chromium, user-managed patched Chrome, or re-attach to a running session. Mirrors `puppeteer.connect`'s shape: `wsEndpoint` for direct WebSocket attachment, `browserURL` for HTTP discovery via `/json/version`. Optional `headers` for auth-gated CDP gateways. `session.close()` disconnects without killing the browser (matches puppeteer's convention via the new `Session.owned` field). See [Connect to an existing Chrome](/docs/guides/connect-existing-chrome).
- **`profile: null` no-spoof mode.** Skip every fingerprint override. Mochi launches a stock Chromium-for-Testing, drives it through the same `Session` / `Page` API surface (humanClick / humanType / session.fetch / cookies / screenshots), but does not derive a `MatrixV1`, compile/splice the inject payload, or send any matrix-keyed CDP override. `Session.profile` surfaces as `null`. Behavioral synthesis falls back to `DEFAULT_BEHAVIOR` from `@mochi.js/behavioral` (`{ hand: "right", tremor: 0.18, wpm: 60, scrollStyle: "smooth" }`).
- **New `@mochi.js/behavioral` export `DEFAULT_BEHAVIOR`.** Distinct from the existing `DEFAULT_BEHAVIOR_PROFILE` (which targets typical typists, `wpm: 65`); `DEFAULT_BEHAVIOR` is the no-spoof fallback (`wpm: 60`) used when the session has no resolved matrix.
- **New low-level CDP transport export.** `connectWebSocketCdp(opts)` + `WebSocketCdpAdapter` interface — escape hatch for users who want to drive raw CDP without the `Session` wrapper.
- **CLI `mochi version` bumped to 0.2.7** to reflect the new `connect`-mode helpers in the programmatic surface.

## v0.8.2 — 2026-05-10

`@mochi.js/core` 0.8.2 + `@mochi.js/inject` 0.4.0.

- **`PerformanceNavigationTiming` zero-handshake leak closed.** New `packages/inject/src/modules/performance-timing.ts`. Chrome launched via `--remote-debugging-pipe` (mochi's launch path) sometimes emitted navigation-timing entries with `domainLookupStart === domainLookupEnd`, `connectStart === connectEnd`, and `nextHopProtocol === ""` even on cold cross-origin loads. Real Chrome on a cold load shows nonzero DNS / TCP times and a populated `nextHopProtocol` (typically `"h2"`). The empty/zero triad is a documented headless tell that FPJS's tampering ML reads. The new module wraps each `PerformanceNavigationTiming` entry in a `Proxy` that overrides `domainLookupEnd`, `connectEnd`, `secureConnectionStart`, `nextHopProtocol` only when their live values are zero/empty; legitimate timing fidelity is preserved. Handshake durations derive from `matrix.uaCh.connection.rtt`. Brings the inject module count from 17 to 18.

## v0.8.1 — 2026-05-10

`@mochi.js/core` 0.8.1 + `@mochi.js/profiles` 0.2.0 + `@mochi.js/consistency` 0.1.4 + `@mochi.js/behavioral` 0.1.5.

- **`@mochi.js/profiles.getProfile()` now actually loads the captured baselines.** Pre-0.8.1 was a `throw new Error("not yet implemented")` stub — every user shipping a string `profile:` got the hardcoded Chrome/131 placeholder UA against an installed Chromium-for-Testing v148. Now reads `data/<id>/profile.json` via `Bun.file()`. New error classes: `UnknownProfileIdError` (id outside `KNOWN_PROFILE_IDS`) and `ProfileBaselineMissingError` (id known but no baseline shipped yet). New `hasProfile(id)` predicate.
- **CfT and placeholder bumped to Chromium 148.** `synthesizePlaceholderProfile` was hardcoded `minVersion: "131"`. `BROWSER_TIP_FULL_VERSION` topped out at `"147"`. `PINNED_FALLBACK_VERSION` was `131.0.6778.85`. Every code path now resolves to `148.0.7778.97` so the spoofed UA major matches the binary.
- **`linux-chrome-stable` profile data fix.** The captured Linux baseline shipped with degraded GPU/display values that read as headless-server (SwiftShader, 1280×800, 32c/64GB, missing branded `sec-ch-ua`). Replaced with realistic Intel Iris Xe values (1920×1080, 8c/16GB, branded `"Google Chrome";v="147"`).
- **Harness conformance picks the host-OS-matched profile.** `CONFORMANCE_PROFILE` was hardcoded `mac-m4-chrome-stable` for every host. Now resolves via `defaultProfileForHost()` so Linux CI gets `linux-chrome-stable`, Mac dev gets `mac-m4-chrome-stable`.

## v0.8.0 — 2026-05-09

`@mochi.js/core` 0.8.0 + `@mochi.js/cli` 0.2.x. The connect-mode + no-spoof-mode + Chromium-148-permission-list wave.

- **`mochi.connect(opts: ConnectOptions): Promise<Session>`** — attach to a CDP browser mochi did NOT spawn. Mirrors `puppeteer.connect`. Supports `wsEndpoint` (direct WebSocket URL) or `browserURL` (HTTP base — mochi GETs `/json/version` to discover the WS URL). `session.close()` disconnects the WebSocket but leaves the browser running. Use cases: BrowserBase, Browserless, dockerised Chromium, re-attach, user-managed patched Chrome. New `ConnectionLostError`. New low-level `connectWebSocketCdp` + `WebSocketCdpAdapter` building blocks. See [Connect to an existing Chrome](/docs/guides/connect-existing-chrome).
- **`profile: null` no-spoof mode** — `mochi.launch({ profile: null })` and `mochi.connect({ profile: null })` skip the matrix derivation, the inject pipeline, and every CDP override (`Network.setUserAgentOverride`, `Emulation.setTimezoneOverride`, `Emulation.setLocaleOverride`, every `addScriptToEvaluateOnNewDocument`). `Session.profile` surfaces as `null`. Behavioral synth falls back to `DEFAULT_BEHAVIOR` (the no-spoof default; distinct from `DEFAULT_BEHAVIOR_PROFILE`). The right shape when you've layered your own stealth elsewhere or want to measure a baseline.
- **`@mochi.js/net` and `@mochi.js/net-rs` deprecated/removed.** Final removal of the Rust `wreq` HTTP layer plumbing. The `wreqPreset` field stays in the schema for back-compat but the runtime ignores it.
- **`ALL_BROWSER_PERMISSIONS` retuned for Chromium 148.** Removed: `accessibilityEvents`, `captureHandle`, `flash`, `videoCapturePanTiltZoom`. Added: `ar`, `vr`, `handTracking`, `automaticFullscreen`, `cameraPanTiltZoom`, `capturedSurfaceControl`, `keyboardLock`, `pointerLock`, `localNetwork`, `localNetworkAccess`, `loopbackNetwork`, `smartCard`, `webPrinting`. 39 entries total.

## v0.7.0 — 2026-05-09

`@mochi.js/core` 0.7.0 drops the Rust `wreq` HTTP layer.

- **`Session.fetch` routes through Chromium itself via CDP.** Simple GETs use `Network.loadNetworkResource` (CORS-bypassed at the network layer); anything with `init` (POST, custom headers, body) uses `page.evaluate("fetch")` against an `about:blank` scratch frame. JA4 / JA3 / H2 are real Chrome by definition because Chromium IS the client — no parallel HTTP layer to keep in lockstep with the spoofed profile, no preset table, no cdylib.
- **Cookie inheritance change.** `Session.fetch` now shares the session's cookie jar with the browser. A cookie set via `Page.goto` or `session.cookies.set` is sent on the next `session.fetch` to the same origin automatically. Pre-0.7 the wreq path was cookieless. Migration: set `init.credentials = "omit"` for the page-evaluate path or clear the relevant cookies before the call.
- **CORS for non-GET.** Mechanism B obeys CORS; cross-origin POSTs without `Access-Control-Allow-Origin` will fail in 0.7 where they may have succeeded in 0.6.
- **Body-shape narrowing.** `Blob` / `FormData` / `ReadableStream` request bodies throw with a clear diagnostic. `string` / `ArrayBuffer` / typed arrays / `URLSearchParams` continue to work.
- **`@mochi.js/net` and `@mochi.js/net-rs` deprecated.** No longer published. The cdylib install friction (`bunx mochi pm trust @mochi.js/net-rs`, the cross-platform prebuild matrix, the `cargo build` fallback) is gone with them.
- **`ProfileV1.wreqPreset` and `MatrixV1.wreqPreset` deprecated.** Runtime no longer reads either field; schema retains them for back-compat.

## v0.6.0 — 2026-05-09

`@mochi.js/core` 0.6.0 — auto-detect Linux server env, default to `--headless=new`, surface a `headlessMode` option.

- **`headlessMode: "new" | "legacy" | "off"`** supersedes the v0.1 `headless: boolean`. `"new"` emits `--headless=new`; `"legacy"` emits bare `--headless`; `"off"` runs headful. The legacy `headless` field is retained — `true` maps to `"new"`, `false` to `"off"`.
- **Linux-server auto-detect.** Snapshots `(process.platform, DISPLAY, WAYLAND_DISPLAY, getuid, container probes)` at launch. Linux without `DISPLAY` / `WAYLAND_DISPLAY` defaults `headlessMode` to `"new"` — closes the "fresh Ubuntu / Debian server" failure mode where a no-DISPLAY box would either crash on first paint or hang attaching to a non-existent display server.
- **`mochi.detectLinuxServerEnv()`** — public introspection helper. Same probe `launch` runs internally.
- **Default profile auto-pick by host OS.** `LaunchOptions.profile` is now optional. When omitted, mochi consults `process.platform` / `process.arch` and auto-picks the host-matching real-device profile. The strategic rationale: spoofing Windows from a Linux server is the wrong default — Linux is a real-user signal in WAFs trained on real traffic, not a bot signal. Production validation: `aone.gg` / FingerprintJS Pro v4 / Linux DC IP / `bot: not_detected` / `suspect_score: 8`. New `defaultProfileForHost()` and `EXPLICIT_PROFILE_IDS` exports.

## v0.4.0 — 2026-05-09

`@mochi.js/core` 0.4.0 — DX cluster + `Page.screenshot` + Fetch.fulfillRequest dual-mech inject.

- **`Session.cookies.{save,load}`** — JSON-backed jar persistence. `CookieJarFile` v1 with `version`, `savedAt`, `mochiVersion`, `pattern`, `count`, `cookies`. Regex domain filter applies on save AND load. JSON, not pickle (Bun-native runtime per PLAN.md I-3).
- **`Page.localStorage.{get,set}`** + **`Page.sessionStorage.{get,set}`** — wrappers around `DOMStorage.getDOMStorageItems` / `setDOMStorageItem`. Frame scope defaults to the page's main-frame origin; pass `{ origin }` to scope explicitly.
- **`Page.grantAllPermissions(opts?)`** — wraps `Browser.grantPermissions` with the full descriptor list. Browser-level grant; page-side `navigator.permissions.query()` is still controlled by the inject (R-036) — orthogonal surfaces.
- **`Page.screenshot`** — PNG / JPEG / WebP via CDP `Page.captureScreenshot`. Options: `format`, `quality`, `fullPage`, `clip`, `omitBackground`, `encoding`. Discriminated overloads narrow the return type by `encoding`.
- **Init-script dual-mechanism delivery.** `Fetch.requestPaused` → `Fetch.fulfillRequest` body splice on Document responses (CSP-rewritten), with `Page.addScriptToEvaluateOnNewDocument({ runImmediately: true, worldName: "" })` as the `about:blank` / `data:` fallback. Closes the source-attribution leak the previous channel carried (the "Vanilla CDP" detection probe) — the inject is byte-indistinguishable from a developer's own `<script>`.
- **Pre-0.4 method shape `Session.cookies(filter)` / `Session.setCookies(...)` removed.** `Session.cookies` is now a getter returning the `CookieJar` namespace.

## v0.3.0 — 2026-05-09

`@mochi.js/core` 0.3.0 — exit-IP / timezone / locale consistency probe + reconciler.

- **`LaunchOptions.geoConsistency: "privacy-fallback" | "auto-correct" | "strict" | "off"`.** Default: `"privacy-fallback"`. Closes the cross-layer leak where `(matrix.timezone, matrix.locale)` and the apparent exit IP disagree.
- **Exit-IP probe.** 7-endpoint registry, shuffled-sequential, 2s per endpoint, 4-attempt cap. Per-endpoint adapter normalises to `ExitGeo`; schema mismatch returns `null` (no throw).
- **`probeExitGeo`, `reconcileGeoConsistency`, `tzOffsetMinutes`, `localeRegion`, `GeoMismatchError`** exported.
